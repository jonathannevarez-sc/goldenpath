# Golden Path — Elevator Pitch

**Version:** v0.3.7

---

## Hook

Every new cloud service shouldn't start from scratch. **Golden Path** is the paved road that gets containerized apps from idea to live on Google Cloud — secure, observable, and deployable in hours, not weeks.

---

## Problem

Today, every new Cloud Run service forces teams to reinvent the same work: repository layout, CI/CD pipelines, IAM, secrets, and monitoring. That costs days or weeks per service, creates inconsistent security postures, and pushes platform decisions onto individual developers. Knowledge lives on laptops and wikis — it drifts between engineers and versions.

---

## Solution

Golden Path is an **enterprise-agnostic platform** — one codebase any organization configures for its own GCP projects and GitHub org. Developers scaffold a service from six templates (Next.js, FastAPI, Streamlit, Express, React SPA, Svelte SPA), push code, and land on Cloud Run with security and observability already wired. Three onboarding paths — CLI, wizard, or AI assistant — all produce the same result. No long-lived keys. No manual GCP console work after scaffold.

---

## Why It Wins

- **Speed:** Under one day to first deploy; zero manual steps after scaffold
- **Consistency:** Same repo shape, pipeline, and conventions every time
- **Security by default:** Keyless auth (WIF), Secret Manager, least-privilege IAM in every template
- **AI-ready:** MCP serves official skills and 13 tools from a pinned release
- **Resilient & opt-in:** CI deploys without MCP; no mandate to migrate legacy systems
